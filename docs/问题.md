# Sitlys 问题核查与修复状态

本文档用于对照历史问题清单与当前代码状态，避免已修问题继续误导后续开发。

## 一、已确认修复

### 1. `cleanURL` 返回值冗余

- 之前 `cleanURL` 存在未使用的第 4 个返回值。
- 当前已改为只返回 `fullURL / host / path` 三项。

### 2. 像素采集缺少 `website_id`

- 之前像素采集入口只传 `pixel_id`，没有显式带 `website_id`。
- 当前 `handleCollectPixel` 已补齐 `website_id`，并支持通过查询参数传 `id` / `vid` 作为访客标识。

### 3. 留存查询全表扫描

- 之前留存分析会先加载整个站点历史 `sessions`，再在 Go 层按时间范围过滤。
- 当前已改为在 SQL 中先限定 cohort 日期范围，并只读取 `cohort_day + 30 天` 内的相关会话。

### 4. Android 平板误判为 mobile

- 之前 Android 设备在 UA 中只要包含 `android` 就会被判成 `mobile`。
- 当前已区分 `android && !mobile` 的情况，按 `tablet` 处理。

### 5. Session Cookie 缺少 `Secure`

- 当前已按请求是否为 HTTPS / `X-Forwarded-Proto=https` 自动设置 `Secure`。

### 6. 事件类型与 PRD 不一致

- 之前代码内部使用 `custom` 作为事件类型，不符合 PRD 中的 `event`。
- 当前已统一为 `event`。

### 7. `pixel` 被计入 `pageviews`

- 之前像素事件会写入 `agg_overview_daily.pageviews` 与 `agg_pages_daily`。
- 当前已修正，只有 `pageview` 才计入页面浏览。

### 8. 页面分析 N+1 查询

- 之前 `handlePages` 会对每个页面路径单独查一次 `count(distinct session_id)`。
- 当前已改为一次 SQL 汇总，再映射回页面列表。

### 9. 过期 Session 不清理

- 当前会在创建新会话登录态时清理已过期 `auth_sessions`。

### 10. 超级管理员把自己降权

- 当前已禁止用户移除自己的 `super_admin` 身份。
- 当前也会阻止把最后一个启用中的 `super_admin` 降级或停用。

### 11. 事件队列满时完全丢弃

- 之前队列满时直接报错，`sendBeacon` 场景下等于静默丢失。
- 当前已增加同步落库回退：队列满时尝试直接写入数据库。

### 12. 采集 API 缺少 CORS

- 当前 `/api/send` 与 `/api/batch` 已支持 `OPTIONS` 预检，并返回基础 CORS 头。

### 13. 接入脚本生成链路不顺

- 之前创建网站后不一定自动选中当前站点，导致脚本区不一定立即可用。
- 当前创建网站后会自动选中该网站，并立即生成对应 `tracker.js` 接入脚本。

## 二、历史结论中已过时或不准确的项

### 1. “漏斗分析使用顺序匹配而非严格有序推进”

- 该结论当前不成立。
- 现有 `runFunnel` 已按事件时间顺序遍历，并按 `stepIndex` 逐步推进。
- 它仍然是“基础顺序匹配”，但不是“步骤 2 可以在步骤 1 之前生效”的那种错误。

### 2. “站点权限可能写入 none”

- 该结论与当前前端实现不一致。
- 现有前端在 `none` 状态下会直接删除该站点权限项，不会把 `access_level: "none"` 提交到后端。
- 但后端仍然值得继续保持防御式校验，这属于增强项，不是当前已发生的问题。

### 3. “设置页是系统配置页却展示统计数据”

- 这是之前前端信息架构问题。
- 当前已收敛：设置页改为系统运行信息、账户安全、运行参数与备份，不再展示网站/像素/分享数量类运营统计。

## 三、仍然存在或尚未完成的问题

### P2

#### 1. 机器人过滤仍然不是可配置体系

- 当前已补充常见爬虫 UA、预取请求识别与基础忽略逻辑。
- 但仍未实现：
  - 后台可配置机器人规则
  - 异常流量审计列表
  - 命中原因报表

#### 2. 地域数据仍依赖上游环境

- 当前已支持从常见 CDN / 反向代理请求头读取 `country / region / city`。
- 但若部署环境不提供这些头、前端也不主动上传，地域报表仍可能为空。

#### 3. 测试覆盖率仍然偏薄

- 当前已进入补测阶段，但整体仍不算完整测试体系。
- 尤其还缺少更细的前端交互校验与更多边界样例。
